## Git 運用ルール

**コードを変更するたびに、必ず GitHub にプッシュすること。**

具体的な手順:

1. 変更を加えたら、こまめに小さな単位でコミットする。
2. コミットメッセージは変更内容が分かるように日本語または英語で簡潔に書く。
3. **コミットした後は、その都度 `git push` で GitHub に反映する。** 変更をローカルに溜め込まない。

```bash
git add -A
git commit -m "変更内容の説明"
git push
```

### 補足

- まだ Git リポジトリが初期化されていない場合は、最初に以下を実行する。

  ```bash
  git init
  git remote add origin <GitHubリポジトリのURL>
  ```

- メインブランチへ直接プッシュするか、ブランチを切って作業するかはプロジェクトの方針に従う。
- 機密情報（APIキー、トークン、パスワード等）はコミットしない。`.gitignore` と `.env` で管理する。
